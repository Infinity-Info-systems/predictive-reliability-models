# Anomaly Detection

## Purpose

Anomaly detection identifies unusual behavior in service metrics, logs, traces, costs, or workflows that may indicate growing reliability risk.

## Detection Areas

- latency drift
- error spikes
- throughput changes
- saturation trends
- cost anomalies
- behavior drift

## Questions To Ask

- what normal looks like?
- what threshold indicates risk?
- what should trigger a review?
- how many false positives are acceptable?

## Evidence To Collect

- anomaly samples
- alert history
- threshold settings
- validation notes

## Use

Use this page to keep anomaly logic tied to real service behavior instead of isolated metric noise.
