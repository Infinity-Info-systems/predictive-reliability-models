# Predictive Alerting Model

## Purpose

Convert predictive signals into alerts that are earlier, more useful, and less noisy.

## Alert Design

- connect to a service outcome
- define a response owner
- include confidence and context
- avoid duplicate alerts
- keep the alert tied to a service outcome

## Output

- alert class
- severity
- response path

## Alert Rule

An alert should earn attention by adding context, not just by repeating a metric breach.
